# Cowork Agent 项目说明书

> 这份文档是给在 Cowork 里接手本项目的 Agent 看的。读完这一页应该能上手。

## 一句话定位

个人自用的"日本买奢侈品包比价 + 现货情报"聚合站，不变现，不对外。核心价值是**多源数据带溯源 + 价差预警 + 现货推送**。

## 用户是谁

中国用户，有买 LV / Chanel 包的习惯，**会自己飞日本买**。痛点是国内 3 万的包日本退税后 2.4 万就能买，但优惠券、店铺差价、现货状况分散在各平台（微博 / X / YouTube / Mercari / 大黑屋 / 小红书），需要一个看板聚到一起。

## 关键决策（已定，别再讨论）

1. **不变现** → 不做用户系统、埋点、合规
2. **数据溯源是硬需求** → 每个价格都必须挂着 `(平台, URL, 作者, 抓取时间, 截图)`
3. **JSON + git 当数据库** → 不上数据库，append-only `observations.jsonl`
4. **GitHub 全免费跑** → Actions cron + Pages 静态站；仓库公开（敏感配置走 Secrets）
5. **UI 选 cute 版** → `mockups/02-cute.html` 是当前视觉基线
6. **小红书优先级低** → 反爬太凶，主战场是微博 + X + Mercari + 品牌官网

## 关键事实（搜过验证过的，别再臆测）

- **Chanel 官网不公开价格** → Chanel 比价基线全靠 UGC，没有"官方爬取"这条路
- **Chanel 款号每季换** → 主键用 `chanel_22bag_medium_black_gold` 这种昵称式 key，不用 SKU
- **三越伊势丹 / 高岛屋的 5% 游客券明确排除 Chanel + LV + VCA** → 别在这两个品牌的 UI 上画 5% 券
- **三越伊势丹 实体卡 2025-03-25 已废止**，改 App 内 coupon
- **Chanel 25 Bag** 是 2025-03-14 真实发布的款，不是凭空编的
- **小红书 RSSHub 路由长期不稳** → 不要把它放在 MVP 必经路径上

## 风险记录

- **小红书反爬**：避免硬碰，用 RSSHub + 半自动
- **Chanel 价格全靠 UGC**：可信度问题，需要在 UI 上展示 AI 置信度 + 多源交叉
- **GitHub Actions 私仓 2000 min/月限额**：仓库要保持 public，敏感配置走 Secrets
- **Mercari 二手鉴别**：上架信息真实但货品真伪需要人工 → 标"参考价"不当推荐价

## 风格 / 价值观（用户自己定的）

- **严谨求实**：不臆测，不确定就说"不确定"
- **不敷衍**：边界、错误路径、并发、性能、回滚都要想
- **诚实交付**：跳过的步骤要在回复里明说

## 沟通风格

用户接受技术细节，不需要包装。直接给方案 + 真实成本 + 风险，让他选。

## 给下一任 Agent 的提醒

- 用户偏好"少做 + 做透" → 别一上来铺很多功能，宁可少而能跑
- 数据溯源是底线，任何"无来源价格"都不能进 `observations.jsonl`
- 涉及钱（变现 / 联盟）的方向用户已明确否，不要再提
- UI 已锁定 cute 版，做技术功能时改 `mockups/02-cute.html` 或它的派生
